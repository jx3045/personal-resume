# Model Selection Proposal: LLM-Assisted Personalized Robust Portfolio Optimization with Smart Rebalancing

## Proposed Section for Team Document

My proposed model direction for the QWIM project is an **LLM-Assisted Personalized Robust Multi-Asset Portfolio Optimization model with Smart Rebalancing**. The model is designed for medium-term and long-term investing, uses daily financial time series, constructs diversified portfolios with approximately 5-20 ETF components, and compares results against clear benchmark portfolios. In this framework, the LLM is not used as a return predictor or as the optimizer itself. Instead, it acts as a personalization and explanation layer that translates investor information into structured model parameters and explains the resulting portfolio in plain language. The quantitative core remains robust portfolio optimization: estimating risk more reliably, applying investor-specific constraints, and controlling turnover through practical rebalancing rules.

The rationale for selecting this model is threefold. First, the model directly matches the main QWIM use case. The project requires multi-asset optimal portfolios for medium- and long-term investors, not short-term trading models. Robust covariance-based portfolio optimization is appropriate for this setting because asset allocation decisions over 3-18 months or longer depend heavily on stable estimates of volatility, correlation, diversification, and drawdown risk. Second, the model is strongly implementable in the existing dashboard backbone. The current code already includes ETF return data, benchmark portfolio data, covariance estimation utilities, convex optimization methods, risk-aversion inputs, portfolio comparison views, and threshold-band rebalancing. Therefore, this model can be implemented as a realistic extension rather than a separate research prototype. Third, it fits the project theme of Personalized Investing + LLMs while avoiding excessive overlap with a Goals-Based Wealth Management model. The LLM component would focus on preference interpretation, parameter recommendation, and explanation, rather than building a full goal-accounting framework.

The proposed modeling pipeline is as follows. First, the LLM reads or interacts with structured and semi-structured client information, such as risk tolerance, investment horizon, liquidity preference, concentration concerns, and rebalancing sensitivity. It converts these inputs into a constrained parameter profile, including risk aversion, maximum asset weight, preferred benchmark, turnover tolerance, and rebalancing band. Second, historical ETF prices are converted into daily returns. A robust covariance estimator is then used to estimate portfolio risk. The baseline version can compare empirical covariance, exponentially weighted covariance, Ledoit-Wolf shrinkage, and oracle approximating shrinkage. The selected model would extend this by emphasizing shrinkage and cross-validation ideas from Tan and Zohren's work on large financial covariance estimation. Third, portfolio weights are obtained using a long-only minimum variance or maximum quadratic utility optimizer. Finally, a smart rebalancing layer decides whether the portfolio should trade back toward target weights, balancing tracking error against transaction costs. The LLM then explains the final allocation, benchmark comparison, and trade-offs in investor-friendly language.

The main references supporting this selection are: Tan and Zohren, **"Estimation of Large Financial Covariances: A Cross-Validation Approach"**, which motivates robust covariance estimation for portfolio construction; De Nard and Kostovic, **"Learning the Shrinkage Intensity for Portfolio Optimization"**, which supports adaptive shrinkage as a future enhancement; Hirukawa, **"Robust Covariance Matrix Estimation in Time Series: A Review"**, which provides theoretical support for robust covariance methods under time-series dependence; Strauss and Williams, **"How to Diversify Differently"**, which motivates testing across different market regimes; and **"Smart Rebalancing"**, which supports the practical layer for turnover-aware portfolio maintenance.

To convince the audience that the selected model is "good enough," I would evaluate it using both performance and implementation evidence. Visuals would include portfolio weight charts, cumulative portfolio value versus benchmark, rolling volatility, rolling drawdown, rolling Sharpe ratio, correlation heatmaps, and turnover/rebalancing event plots. Metrics would include annualized return, annualized volatility, Sharpe ratio, Sortino ratio, maximum drawdown, Calmar ratio, turnover, transaction-cost-adjusted return, tracking error, and concentration measures. Testing periods should include multiple horizons, such as 1-year, 3-year, 5-year, 10-year, and all available history, with special attention to stressful regimes such as 2020 and 2022. Benchmarks should include equal weight, 60/40 or the provided benchmark portfolio, risk parity, maximum diversification, and the current dashboard optimization methods.

The expected dashboard implementation would be an independent subtab named something like **LLM-Assisted Personalized Optimization**. The subtab would allow the user to choose or generate a risk profile, covariance estimator, asset universe, time period, benchmark, and rebalancing rule. The output would show optimized weights, benchmark comparison, risk metrics, rebalancing diagnostics, and an LLM-generated explanation of why the portfolio fits the selected investor profile. This makes the model not only theoretically justified but also demonstrable inside the QWIM dashboard.

The main implementation risks are manageable. A full adaptive machine-learning shrinkage model may require more training infrastructure, so I would treat it as a second-stage enhancement. Similarly, the first-stage LLM component should be limited to structured parameter extraction and explanation, with validation rules to prevent unreasonable outputs. The first-stage model should focus on robust shrinkage covariance, long-only optimization, risk-tolerance-based constraints, threshold/smart rebalancing, and LLM-generated interpretability. This version is academically defensible, technically feasible, aligned with Personalized Investing + LLMs, and sufficiently distinct from teammates' goal-based models.
