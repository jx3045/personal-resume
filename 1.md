# Questions for Course Coordinator — QWIM Project

As our team starts working on the QWIM project, we have a few practical questions about team/individual responsibilities, mentor interactions, and the LLM component. We'd appreciate your clarification on the following.

1. Can two members within the same team choose models from the same category (e.g., both picking from Machine Learning), or should each member cover a different category for breadth?

2. If a member finds their chosen model too difficult to implement after starting (e.g., no available code, overly complex math), is it possible to switch to a different model mid-project? What's the process for that?

3. Regarding the final report — should each member write their own sections (literature review, model description, numerical results) separately, while the team collaborates on the Executive Summary, comparative analysis, and Conclusion? Or is the entire report expected to be written as one unified document?

4. Will there be a peer evaluation component where team members rate each other's contributions? If so, how does it affect individual grades?

5. Should all communication with Cristian go through one team representative, or can each member reach out independently about their own model?

6. If Cristian suggests changes to our approach that we disagree with, does the team have final say, or is his approval required before proceeding?


7. What is the minimum expected level of LLM integration for each member's work? Is it enough for LLMs to play a supporting role (e.g., parsing natural language goals, generating explanations), or do they need to be more central to the investment decision process?

8. If a member's chosen model doesn't naturally connect with LLMs (e.g., a purely mathematical model), can the LLM component be handled at the team level as a shared module, or does each member need their own LLM integration?

9. Does the LLM integration need to be a live, working feature inside the Dashboard (e.g., a chatbot or real-time text generation), or is a conceptual description with offline demonstration sufficient for the report and final evaluation?
