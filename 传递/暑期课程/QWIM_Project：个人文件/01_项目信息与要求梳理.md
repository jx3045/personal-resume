## 五、项目要求详解

### 5.1 文献综述阶段（前2-4周）

项目建议前3-4周将**文献综述**与**数据集检索**结合进行。文献综述采用**分阶段方法**：

- **第一阶段：** 仅基于论文摘要（Abstract），决定保留哪些文献进入下一阶段
- **第二阶段：** 阅读保留文献的摘要、结论（Conclusion）和数值结果（Numerical Results）三个部分，决定继续保留哪些
- **第三阶段：** 对保留文献，判断其模型/方法是否适用于本项目的用例；如有疑问可联系论文作者
- **第四阶段：** 最终比较并选出2-5篇最相关的文献

**参考文献阅读顺序建议：**
1. 优先阅读 Section 1.2: Top recommendations within main references
2. （可选）Section 1.1: Main references
3. （可选）Section 1.3: Comprehensive list of references

### 5.2 模型选择

- 每位团队成员需选择至少**1个量化模型/方法**
- 模型选择需与导师（Cristian）和学术导师（Academic Advisor）沟通确认
- 需要提供**选择该模型的理由和支持证据**，说明为何该模型在本项目场景下有良好潜力
- 模型可来自导师提供的文献列表或自行检索的相关文献

### 5.3 核心使用场景（Main Use Case）

在选择模型和文献时，始终围绕以下使用场景：

- **中期投资（3-18个月）和/或长期投资（18个月以上）**
- **多资产最优投资组合**：至少3个组成部分（推荐5-20个）
- **金融时间序列**：使用最多为日频（daily）的金融收益数据
- **与精心选择的基准（Benchmark）进行比较**

> 注意：许多模型可能适用于短期交易（日内数据），但本项目只考虑对中长期投资场景有良好潜力的方法。

### 5.4 编码与开发要求

**推荐编码环境：**
- Visual Studio Code (VSC) + 扩展（包括Python扩展）、Coding Agents
- Python版本：3.13.13 或更高
- 包管理：使用 `uv` 进行项目和包管理
- 数据处理：推荐使用 `polars`（而非pandas）
- 自动化框架：推荐使用 `kedro`
- 代码质量：使用 `ruff` 进行代码格式化

**环境搭建命令：**
```bash
python pip install -U pip uv ruff
uv add polars
pytest tests -n auto -q          # 运行测试
shiny run --launch-browser src/dashboard/main_App.py   # 运行Dashboard
```

**GitHub仓库要求：**
- 创建**私有（Private）**GitHub仓库
- 邀请 AlexPars（导师的GitHub账户）加入
- README文件需包含：团队名称、项目名称、成员姓名和邮箱
- 遵循软件开发最佳实践：使用Feature Branching、Pull Request、更新测试

**Dashboard要求：**
- 推荐使用 **Shiny（Python或R版）**，次选 Streamlit
- 导师会在私有GitHub仓库中提供Shiny Dashboard的Python代码
- 每个模型作为Dashboard的**独立Tab**（含所需的Sub-tab）
- 每个模型需连接到客户端输入（已在Dashboard的 `Clients` Tab中实现）

### 5.5 团队编码框架结构

**个人组件（Individual Components）：**
- 每位成员选择的模型/方法的独立实现

**公共组件（Common Components）：**

## 六、预期产出与报告架构

### 6.1 最终交付物

本项目需要交付**两项核心成果**：

1. **书面报告（Written Report）**
2. **交互式仪表盘（Interactive Dashboard）**

### 6.2 报告架构

最终报告的结构如下：

1. **Executive Summary（执行摘要）** — 1-2页
2. **Introduction（引言）**
3. **Literature Review and Analysis（文献综述与分析）**
4. **Description of Selected Models（所选模型描述）**
5. **Numerical Results（数值结果）**
6. **Analysis of Results including Narratives（结果分析，含叙述性解读）**
   - 解释结果的含义
   - 尽可能将结果与金融市场行为、经济状况、新闻事件、地缘政治事件等联系起来
7. **Conclusion（结论）**
8. **List of References（参考文献列表）**
9. **Appendices（附录，如需要）**

**说明：** 该报告在形式上类似于一篇**学术研究论文（Research Report）**，具有完整的学术论文结构。虽然不是严格意义上的期刊论文投稿，但其结构、深度和严谨性参照学术论文标准。报告需提交给学术导师（Academic Advisor）作为QWIM项目的最终评估依据。

### 6.3 项目重点强调

项目非常强调**"模型选择/实现 + 说服受众认可所选模型的优势和实用性"**的结合，而不仅仅是模型实现本身。这也是要求开发交互式Dashboard的原因——通过可视化和交互功能展示模型效果。
